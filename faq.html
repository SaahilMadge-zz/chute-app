Frequently Asked Questions


What browsers does Chute work on?
Chute currently only works on Chrome (tested on v34+). Support for Firefox and Opera is in the works. Unfortunately, due to technical limitations with the WebRTC API, we currently do not have plans to implement Chute on Internet Explorer or Safari.
Why do I need cookies enabled?
Chute uses cookies to detect dropped connections and attempt to reconnect users. Without cookies, it would be impossible to determine whether a returning user was the same one who disconnected.
Why can't I send a file to myself?
We create a cookie for both the sender and the receiver, but if the sender and receiver are both the same browser then the application will be unable to create the second cookie. This causes the application to hang.
Is the file size really unlimited?
Not quite. Chrome allows each application to use 10% of your total available HDD space. For example, if your hard drive has 28 GB of free space you will be able to receive a file of at most 2.8 GB.
Can I use Chute in incognito mode?
Sort of. Because of security issues, Chrome does not allow applications to store local data in private browsing mode, which is necessary for receiving files. You can, however, send files from incognito mode.
Are there any legal issues with using Chute?
Chute is meant for sending files that can legally be transferred. Sending a file on Chute is no functionally different than sending a file over email, except that Chute does not host the data transferred between two users. All users are subject to the same copyright laws and regulations that govern the sharing of all data over the Internet. Failure to adhere to these laws may result in legal action.
